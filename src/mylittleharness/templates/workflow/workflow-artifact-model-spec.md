# Workflow Artifact Model Spec

This stable workflow spec fixture anchors the expected artifact model surface for a MyLittleHarness operating root.

Repo-visible project files remain authoritative. This fixture is repair-created only when absent, and it cannot approve repair, closeout, archive, commit, rollback, lifecycle decisions, or future mutations.

Agent governance evidence stays repo-visible and report-only by default. Work claims, approval packets, agent-run records, review tokens, `reconcile`, `incubation-reconcile`, and `check --focus agents` can expose fingerprints, stale claims, drift, missing evidence, incubation fate metadata, and worker-space residue, but they do not create hidden runtime state, clean worktrees, rewrite authority, archive plans, or approve lifecycle movement.
