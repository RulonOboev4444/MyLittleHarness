# Workflow Capability Roadmap Spec

This stable workflow spec fixture anchors the expected capability roadmap surface for a MyLittleHarness operating root.

Repo-visible project files remain authoritative. This fixture is repair-created only when absent, and it cannot approve repair, closeout, archive, commit, rollback, lifecycle decisions, or future mutations.

The optional live-root roadmap route is `project/roadmap.md`. It is a human-readable sequencing surface for accepted work between incubation and one active implementation plan; it is not part of the mandatory start path and cannot approve repair, closeout, archive, commit, rollback, or lifecycle decisions.

Roadmap item metadata should stay minimal and frontmatter-friendly: stable `id`, `status`, advisory `order` or `priority`, `execution_slice`, `slice_goal`, `slice_members`, `slice_dependencies`, `slice_closeout_boundary`, `dependencies`, `source_incubation`, `source_research`, `related_specs`, `related_plan`, `archived_plan`, `target_artifacts`, `verification_summary`, `docs_decision`, `carry_forward`, `supersedes`, and `superseded_by`. Read-only diagnostics may validate these route relationships when the file exists, the hidden `roadmap --dry-run|--apply` rail may add one explicit canonical item block, update one canonical item block or existing compatibility top-level `## RM-...` section, compact completed MLH-owned Future Execution Slice Queue bullets, and compact older detailed canonical `done` item blocks with archived-plan evidence into Archived Completed History in an eligible live operating root, hidden `incubate --apply` may record structurally known active-plan and active-plan roadmap relationship metadata in the target incubation note, and hidden `plan --roadmap-item` / `writeback --roadmap-item` may sync requested roadmap relationships plus active-plan or roadmap-slice `covered_roadmap_items` without making roadmap output lifecycle authority. Compatibility RM sections can keep work-area `slice_members` labels without expanding them as covered roadmap ids. Plan opening also refreshes each explicit source-incubation note back to the new active plan, and reviewed transition archive skips the next roadmap item's source note before opening that next plan. If a writeback source incubation already lives under `project/archive/reference/incubation/**`, the relationship sync updates it in place and does not re-archive it.

Relationship metadata stays path/id based: source incubation/research, related roadmap item, active or archived plan, specs, decisions, ADRs, verification, archive, supersession, merge, split, and rejection links are readable Markdown/frontmatter fields, not a separate authoritative index. Generated relationship graphs such as `.mylittleharness/generated/projection/relationships.json` are rebuildable navigation projections only. Whole-file incubation auto-archive is safe only for structurally covered notes: single-entry notes can be covered by the closeout chain, while mixed notes require terminal `## Entry Coverage` bullets for every dated entry. Mixed or open-thread notes without that coverage remain active and receive blocker diagnostics plus heuristic split suggestions. Implemented incubation note archives use `project/archive/reference/incubation/**`.
